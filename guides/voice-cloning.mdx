## Overview

Voice cloning in Duix Avatar uses advanced text-to-speech (TTS) technology to create a digital voice model from audio samples. The system supports 8 languages and provides high-quality voice synthesis for your avatars.

## How Voice Cloning Works

The voice cloning process involves two main steps:
1. **Preprocessing and Transcription**: Audio is analyzed and transcribed
2. **Voice Model Creation**: A unique voice profile is generated

## The Training Process

<Steps>
  <Step title="Audio Preprocessing">
    The `train` function initiates the voice training process:

    ```javascript
    // From src/main/service/voice.js:17
    export async function train(path, lang = 'zh') {
      // Normalize path separators
      path = path.replace(/\\/g, '/') 
      
      // Call preprocessing API
      const res = await preprocessAndTran({
        format: path.split('.').pop(),
        reference_audio: path,
        lang
      })
      
      if (res.code !== 0) {
        return false
      }
      
      // Save voice model to database
      const { asr_format_audio_url, reference_audio_text } = res
      return insert({ 
        origin_audio_path: path, 
        lang, 
        asr_format_audio_url, 
        reference_audio_text 
      })
    }
    ```
  </Step>

  <Step title="API Processing">
    The preprocessAndTran API analyzes your audio:
    
    ```javascript
    // From src/main/api/tts.js:12
    export function preprocessAndTran(param) {
      return request.post(`${serviceUrl.tts}/v1/preprocess_and_tran`, param)
    }
    ```
    
    **Parameters:**
    - `format`: Audio file format (wav, mp3, etc.)
    - `reference_audio`: Path to the audio file
    - `lang`: Language code (zh, en, ja, ko, de, fr, es, it)
    
    **Returns:**
    - `asr_format_audio_url`: Processed audio URL
    - `reference_audio_text`: Transcribed text from the audio
  </Step>

  <Step title="Voice Model Storage">
    After successful training, the voice model is stored in the database:
    
    ```javascript
    insert({ 
      origin_audio_path: path,           // Original audio location
      lang,                               // Language code
      asr_format_audio_url,              // Processed audio URL
      reference_audio_text                // Transcribed text
    })
    ```
    
    A unique voice ID is returned for future use.
  </Step>
</Steps>

## Generating Audio with Cloned Voice

Once a voice is trained, you can generate audio using the `makeAudio` function:

```javascript
// From src/main/service/voice.js:46
export async function makeAudio({voiceId, text, targetDir}) {
  const uuid = crypto.randomUUID()
  const voice = selectByID(voiceId)

  return makeAudioApi({
    speaker: uuid,
    text,
    format: 'wav',
    topP: 0.7,
    max_new_tokens: 1024,
    chunk_length: 100,
    repetition_penalty: 1.2,
    temperature: 0.7,
    need_asr: false,
    streaming: false,
    is_fixed_seed: 0,
    is_norm: 1,
    reference_audio: voice.asr_format_audio_url,
    reference_text: voice.reference_audio_text
  })
}
```

## Voice Generation Parameters

The TTS API accepts several parameters to control voice generation:

| Parameter | Value | Description |
|-----------|-------|-------------|
| `topP` | 0.7 | Nucleus sampling parameter |
| `max_new_tokens` | 1024 | Maximum length of generated audio |
| `chunk_length` | 100 | Text chunk size for processing |
| `repetition_penalty` | 1.2 | Reduces repetitive speech patterns |
| `temperature` | 0.7 | Controls randomness in generation |
| `is_norm` | 1 | Audio normalization enabled |
| `format` | wav | Output audio format |

<Note>
These parameters are optimized for quality and can be adjusted based on your specific needs.
</Note>

## Voice Audition

You can preview a cloned voice before using it in video synthesis:

```javascript
// From src/main/service/voice.js:87
export async function audition(voiceId, text) {
  const tmpDir = require('os').tmpdir()
  const audioPath = await makeAudio({ 
    voiceId, 
    text, 
    targetDir: tmpDir 
  })
  return path.join(tmpDir, audioPath)
}
```

## Using Custom Audio Files

For video synthesis, you can also use custom audio files instead of generating TTS:

```javascript
// From src/main/service/voice.js:37
export function copyAudio4Video(filePath) {
  const targetDir = assetPath.ttsProduct
  const fileName = dayjs().format('YYYYMMDDHHmmssSSS') + path.extname(filePath)
  const targetPath = path.join(targetDir, fileName)
  fs.copyFileSync(filePath, targetPath)
  return fileName
}
```

This is useful when:
- You have pre-recorded professional audio
- You want precise control over speech timing
- You need to use audio edited in external tools

## Best Practices

### Audio Quality

<Tip>
**For optimal voice cloning results:**
- Use high-quality audio (16-bit, 44.1kHz or higher)
- Ensure minimal background noise
- Include varied speech patterns and emotions
- Provide 30+ seconds of clear speech
</Tip>

### Language Considerations

<Warning>
Always match the language parameter to the actual language spoken in the audio. Mismatched languages will result in poor transcription and voice quality.
</Warning>

### Storage and Organization

1. **Voice Database**: All trained voices are stored in the database with their associated audio URLs
2. **Original Audio**: Keep original audio files for potential retraining
3. **Generated Audio**: TTS-generated audio is stored in `assetPath.ttsProduct`
4. **Training Audio**: Training samples are stored in `assetPath.ttsTrain`

## Troubleshooting

### Training Fails

If voice training fails (`res.code !== 0`):
1. Check audio file format and quality
2. Verify language parameter is correct
3. Ensure audio file path is accessible
4. Check network connection to TTS service

### Poor Voice Quality

1. **Use longer audio samples**: More data improves quality
2. **Reduce background noise**: Clean audio produces better models
3. **Check language match**: Ensure audio language matches the parameter
4. **Verify audio format**: WAV format provides best results

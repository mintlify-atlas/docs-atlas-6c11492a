## Overview

Duix Avatar requires an NVIDIA GPU with properly configured drivers and the NVIDIA Container Toolkit for Docker to access GPU resources. All three services (TTS, ASR, and video generation) depend on CUDA for acceleration.

## Hardware Requirements

### Supported GPUs

<CardGroup cols={3}>
  <Card title="RTX 30 Series" icon="microchip">

  <Card title="RTX 40 Series" icon="microchip">

  </Card>
  <Card title="RTX 50 Series" icon="microchip">
    RTX 5090 (Optimized images available)
  </Card>

### Recommended Configuration

- **GPU**: NVIDIA RTX 4070 or better
- **VRAM**: 12GB minimum, 16GB+ recommended
- **CUDA**: Version 11.8 or higher (12.8 for RTX 50 series)
- **Driver**: Latest NVIDIA driver for your GPU series

<Note>
RTX 50 series GPUs require CUDA 12.8 and use special Docker images with PyTorch preview builds.
</Note>

## NVIDIA Driver Installation

### Windows

1. Visit the [NVIDIA Driver Downloads](https://www.nvidia.com/drivers/lookup/) page

2. Select your GPU model and operating system:
   - Product Type: GeForce
   - Product Series: RTX 40 Series (or your series)
   - Product: RTX 4070 (or your model)
   - Operating System: Windows 10/11 64-bit

3. Download and run the installer

4. Choose "Custom" installation and select "Clean Installation" for best results

5. Restart your computer after installation

6. Verify installation:

```bash
nvidia-smi
```

Expected output:

```
+-----------------------------------------------------------------------------+
| NVIDIA-SMI 560.94       Driver Version: 560.94       CUDA Version: 12.6     |
|-------------------------------+----------------------+----------------------+
| GPU  Name            TCC/WDDM | Bus-Id        Disp.A | Volatile Uncorr. ECC |
| Fan  Temp  Perf  Pwr:Usage/Cap|         Memory-Usage | GPU-Util  Compute M. |
|===============================+======================+======================|
|   0  NVIDIA GeForce ... WDDM  | 00000000:01:00.0  On |                  N/A |
| 30%   45C    P8    15W / 200W |    512MiB / 12288MiB |      2%      Default |
+-------------------------------+----------------------+----------------------+
```

### Linux (Ubuntu)

#### Option 1: Using Ubuntu's Built-in Driver Manager

1. Open "Software & Updates" → "Additional Drivers" tab

2. Select the recommended NVIDIA driver (proprietary, tested)

3. Click "Apply Changes" and restart

#### Option 2: Manual Installation

1. Add the NVIDIA driver PPA:

```bash
sudo add-apt-repository ppa:graphics-drivers/ppa
sudo apt update
```

2. Search for available drivers:

```bash
apt search nvidia-driver
```

3. Install the recommended driver:

```bash
sudo apt install nvidia-driver-560
```

4. Reboot your system:

```bash
sudo reboot
```

5. Verify installation:

```bash
nvidia-smi
```

<Warning>
If `nvidia-smi` returns an error, the driver installation failed. Check `/var/log/syslog` for errors.
</Warning>

## NVIDIA Container Toolkit Setup

The NVIDIA Container Toolkit allows Docker containers to access GPU resources.

### Windows

Docker Desktop for Windows automatically includes NVIDIA Container Toolkit support when Docker is configured to use WSL 2 and NVIDIA drivers are installed.

**Verify GPU access in Docker:**

```bash
docker run --rm --gpus all nvidia/cuda:11.8.0-base-ubuntu22.04 nvidia-smi
```

If this command displays GPU information, your setup is complete.

### Linux

#### 1. Add NVIDIA Package Repository

```bash
distribution=$(. /etc/os-release;echo $ID$VERSION_ID) \
  && curl -s -L https://nvidia.github.io/libnvidia-container/gpgkey | sudo apt-key add - \
  && curl -s -L https://nvidia.github.io/libnvidia-container/$distribution/libnvidia-container.list | sudo tee /etc/apt/sources.list.d/nvidia-container-toolkit.list
```

#### 2. Install NVIDIA Container Toolkit

```bash
sudo apt-get update
sudo apt-get install -y nvidia-container-toolkit
```

#### 3. Configure Docker Runtime

Configure Docker to use the NVIDIA runtime:

```bash
sudo nvidia-ctk runtime configure --runtime=docker
```

This command modifies `/etc/docker/daemon.json` to include:

```json
{
  "runtimes": {
    "nvidia": {
      "path": "nvidia-container-runtime",
      "runtimeArgs": []
    }
  }
}
```

#### 4. Restart Docker Service

```bash
sudo systemctl restart docker
```

#### 5. Verify GPU Access

Test GPU access from Docker:

```bash
docker run --rm --gpus all nvidia/cuda:11.8.0-base-ubuntu22.04 nvidia-smi
```

Expected output should show your GPU information.

## GPU Resource Allocation

### Environment Variables

Duix Avatar services use NVIDIA environment variables to control GPU access:

#### NVIDIA_VISIBLE_DEVICES

Controls which GPUs are visible to the container:

```yaml
environment:
  - NVIDIA_VISIBLE_DEVICES=0  # Use GPU 0 only
  # - NVIDIA_VISIBLE_DEVICES=all  # Use all GPUs
  # - NVIDIA_VISIBLE_DEVICES=0,1  # Use GPU 0 and 1
```

#### NVIDIA_DRIVER_CAPABILITIES

Specifies which driver capabilities to enable:

```yaml
environment:
  - NVIDIA_DRIVER_CAPABILITIES=compute,graphics,utility,video,display
```

Capabilities:
- `compute`: CUDA and OpenCL
- `graphics`: OpenGL and Vulkan
- `utility`: nvidia-smi and other utilities
- `video`: Video encoding/decoding
- `display`: Display capabilities

### Docker Compose GPU Configuration

#### Method 1: Runtime + Environment (TTS Service)

```yaml
duix-avatar-tts:
  runtime: nvidia
  environment:
    - NVIDIA_VISIBLE_DEVICES=0
    - NVIDIA_DRIVER_CAPABILITIES=compute,graphics,utility,video,display
```

#### Method 2: Deploy Resources (ASR and Video Services)

```yaml
duix-avatar-asr:
  deploy:
    resources:
      reservations:
        devices:
          - driver: nvidia
            count: all  # Use all available GPUs
            capabilities: [gpu]
```

```yaml
duix-avatar-gen-video:
  deploy:
    resources:
      reservations:
        devices:
          - capabilities: [gpu]
```

### Memory Management

The video generation service uses PyTorch memory optimization:

```yaml
environment:
  - PYTORCH_CUDA_ALLOC_CONF=max_split_size_mb:512
```

This limits CUDA memory fragmentation by setting the maximum split size to 512MB.

### Shared Memory

The video service requires 8GB of shared memory for efficient operation:

```yaml
shm_size: '8g'
```

<Warning>
Ensure your system has sufficient RAM available. The 8GB shared memory is in addition to regular container memory.
</Warning>

## RTX 50 Series Support

### Special Configuration for RTX 5090

The RTX 50 series requires different Docker images with CUDA 12.8 support:

<CodeGroup>

```yaml TTS Service (5090)
duix-avatar-tts:
  image: guiji2025/fish-speech-5090  # Note: -5090 suffix
  container_name: duix-avatar-tts
  runtime: nvidia
  working_dir: /code
  environment:
    - NVIDIA_VISIBLE_DEVICES=0
    - NVIDIA_DRIVER_CAPABILITIES=compute,graphics,utility,video,display
  ports:
    - '18180:8080'
  volumes:
    - d:/duix_avatar_data/voice/data:/code/data
  command: /bin/bash -c "python tools/api_server.py --listen 0.0.0.0:8080"
```

```yaml Video Service (5090)
duix-avatar-gen-video:
  image: guiji2025/duix.avatar-5090  # Note: -5090 suffix
  container_name: duix-avatar-gen-video
  runtime: nvidia
  privileged: true
  volumes:
    - d:/duix_avatar_data/face2face:/code/data
  environment:
    - PYTORCH_CUDA_ALLOC_CONF=max_split_size_mb:512
  deploy:
    resources:
      reservations:
        devices:
          - capabilities: [gpu]
  shm_size: '8g'
  ports:
    - '8383:8383'
  command: python /code/app_local.py
```

</CodeGroup>

### RTX 50 Series Features

- **CUDA 12.8** support
- **PyTorch Preview** builds with RTX 50 optimizations
- **Backward compatible** with RTX 30/40 series
- **No ASR service** in 5090 configuration

<Info>
The RTX 5090 images have been tested and verified on RTX 5090 GPUs. They also work with RTX 30/40 series cards.
</Info>

## Verification Steps

### 1. Check NVIDIA Driver

```bash
nvidia-smi
```

Verify:
- Driver version is displayed
- CUDA version is shown
- GPU is listed with correct name
- No errors are reported

### 2. Test Docker GPU Access

```bash
docker run --rm --runtime=nvidia --gpus all nvidia/cuda:11.8.0-base-ubuntu22.04 nvidia-smi
```

Should display the same GPU information as the host system.

### 3. Verify Duix Avatar Services

Start the services and check GPU usage:

```bash
# In one terminal
docker-compose up -d

# In another terminal
watch -n 1 nvidia-smi
```

You should see GPU memory usage increase as services start.

### 4. Check Service Logs

```bash
docker logs duix-avatar-tts | grep -i cuda
docker logs duix-avatar-asr | grep -i cuda
docker logs duix-avatar-gen-video | grep -i cuda
```

Look for CUDA initialization messages and successful GPU detection.

## Common GPU Issues

### Issue: "nvidia-smi: command not found"

**Solution**: NVIDIA drivers are not installed or not in PATH.

- Windows: Reinstall NVIDIA drivers
- Linux: Install drivers using the steps above

### Issue: "NVIDIA-SMI has failed because it couldn't communicate with the NVIDIA driver"

**Solution**: Driver is installed but not loaded.

- Windows: Restart the system
- Linux: Run `sudo modprobe nvidia` and check `dmesg | grep -i nvidia` for errors

### Issue: Docker containers can't access GPU

**Solution**: NVIDIA Container Toolkit not configured.

- Windows: Ensure WSL 2 is being used and Docker Desktop is updated
- Linux: Install NVIDIA Container Toolkit using the steps above

### Issue: "could not select device driver with capabilities: [[gpu]]"

**Solution**: Docker daemon not configured for NVIDIA runtime.

```bash
sudo nvidia-ctk runtime configure --runtime=docker
sudo systemctl restart docker
```

### Issue: Out of memory errors

**Solution**: Insufficient GPU memory.

1. Close other GPU-intensive applications
2. Use a GPU with more VRAM (16GB+ recommended)
3. Adjust `PYTORCH_CUDA_ALLOC_CONF` settings

## Performance Optimization

### Multi-GPU Setup

If you have multiple GPUs, you can distribute services:

```yaml
duix-avatar-tts:
  environment:
    - NVIDIA_VISIBLE_DEVICES=0  # Use GPU 0

duix-avatar-asr:
  environment:
    - NVIDIA_VISIBLE_DEVICES=1  # Use GPU 1

duix-avatar-gen-video:
  environment:
    - NVIDIA_VISIBLE_DEVICES=0  # Use GPU 0 (or 1)
```

### GPU Monitoring

Monitor GPU usage in real-time:

```bash
watch -n 1 nvidia-smi
```

Or use NVIDIA's detailed monitoring:

```bash
nvidia-smi dmon -s pucvmet
```

## Reporting Issues

When reporting bugs or issues, help us help you by providing detailed information:

### Before Creating an Issue

<Steps>
  <Step title="Search Existing Issues">
    Check if your issue has already been reported or solved:
    - [Search GitHub Issues](https://github.com/duixcom/Duix-Avatar/issues?q=is%3Aissue)
    - Many issues are already documented with solutions
    - Check closed issues too - your problem may be fixed in a newer version
  </Step>

  <Step title="Complete Self-Check Steps">
    Verify the basics before reporting:
    
    1. **Confirm all three services are running:**
       ```bash
       docker ps
       ```
       You should see: `duix-avatar-tts`, `duix-avatar-asr`, `duix-avatar-gen-video`
    
    2. **Verify NVIDIA GPU is detected:**
       ```bash
       nvidia-smi
       ```
    
    3. **Update to latest version:**
       ```bash
       # Server
       cd deploy
       docker-compose pull
       docker-compose up -d
       
       # Client: Download from GitHub releases
       ```
    
    See the [FAQ](/support/faq) for complete self-check instructions.
  </Step>

  <Step title="Collect Error Information">
    Gather relevant logs and error messages:
    
    **Client Logs:**
    - Open Duix Avatar app
    - Navigate to Settings → View Logs
    - Copy the latest log file
    
    **Server Logs:**
    ```bash
    # Save logs for each service
    docker logs duix-avatar-tts > tts-logs.txt 2>&1
    docker logs duix-avatar-asr > asr-logs.txt 2>&1
    docker logs duix-avatar-gen-video > video-logs.txt 2>&1
    ```
    
    **System Information:**
    - Operating System (Windows 10/11, Ubuntu 22.04, etc.)
    - GPU model (RTX 3090, RTX 4070, etc.)
    - NVIDIA driver version (from `nvidia-smi`)
    - Docker version (from `docker --version`)
    - Client version (from Help → About)
  </Step>
</Steps>

### Creating a Good Bug Report

Use this template when creating a GitHub issue:

```markdown
## Description
[Clear description of the problem]

## Steps to Reproduce
1. [First step]
2. [Second step]
3. [Third step]

## Expected Behavior
[What you expected to happen]

## Actual Behavior
[What actually happened]

## Environment
- OS: [Windows 11 / Ubuntu 22.04 / etc.]
- GPU: [RTX 4070 / RTX 3090 / etc.]
- NVIDIA Driver: [535.129.03 / etc.]
- Docker Version: [24.0.6 / etc.]
- Client Version: [1.0.6 / etc.]

## Logs
[Attach or paste relevant error logs]

## Screenshots
[If applicable, add screenshots to help explain the problem]

## Additional Context
[Any other information that might be helpful]
```

<Note>
The more information you provide, the faster the community can help you solve the issue.
</Note>

## Feature Requests

Have an idea to improve Duix Avatar? We welcome feature requests!

<Steps>
  <Step title="Check Existing Requests">
    Search [GitHub Issues](https://github.com/duixcom/Duix-Avatar/issues?q=is%3Aissue+label%3Aenhancement) with the "enhancement" label to see if someone already suggested it.
  </Step>

  <Step title="Describe Your Use Case">
    When creating a feature request, explain:
    - What problem you're trying to solve
    - Why existing features don't meet your needs
    - How you envision the feature working
    - Who would benefit from this feature
  </Step>

  <Step title="Be Open to Discussion">
    The community may suggest alternative approaches or workarounds. Be open to feedback and collaboration.
  </Step>
</Steps>

## Community Guidelines

When participating in the Duix Avatar community:

✅ **Do:**
- Be respectful and constructive
- Search before asking questions
- Provide detailed information when reporting issues
- Share your solutions to help others
- Give credit where due

❌ **Don't:**
- Spam or post off-topic content
- Demand immediate responses
- Share others' private information
- Use offensive or discriminatory language
- Hijack existing threads for unrelated issues
